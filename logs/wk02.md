## What I Learned

### HTML Basics

- The structure of an HTML file makes more sense now that I typed it out myself. DOCTYPE html, html, head, body — it is like a skeleton and everything goes inside it
- The head section is kind of invisible — it holds the title and other info but nothing from there actually shows up on the page
- Browsers are weirdly forgiving with broken HTML, which sounds nice but actually makes it harder to find mistakes

### Images Were Confusing

- The file path in src has to match exactly — if the file is .png and you write .jpg it just will not show up and you will not always know why
- I had the photo open in VS Code from my Downloads and thought that counted — it does not, it has to actually be inside the project folder
- You can resize with width="400" right in the tag which is simple once you know it

### VS Code and Live Server

- Live Server is genuinely useful — every time I saved, the browser just updated with no extra steps
- The file tree in the sidebar is your actual folder structure, not just a view — dragging files in there actually moves them into the project

### Copilot and AI Tools

- Started learning how to use GitHub Copilot — it gives suggestions as you type and you just hit Tab to accept
- It is helpful but I had to be careful not to just accept everything without reading it
- Also learned that how you phrase a prompt matters a lot — vague questions get vague answers
